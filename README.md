# 정보프로그래밍심화 - 중간 프로젝트

연세대학교 캠퍼스를 배경으로 한 텍스트 어드벤처 게임.
주인공이 학교를 탐험하며 임무를 해결하는 콘솔 기반 RPG입니다.

## 📖 게임 소개

송도 생활을 마치고 신촌에 처음 도착한 주인공은 정문에서 시작하여 학교 곳곳을 탐험합니다.
독수리상에서 임무를 받고, 부조리와 식중독 사건을 조사하여 본관과 세브란스에 보고한 뒤,
이윤재관에서 수업을 듣고 게임을 종료합니다.

## 🚀 실행 방법

### 일반 실행

```bash
python main.py
```

### 교수님이 주신 `run.py`로 실행 (채점 환경과 동일)

```bash
python run.py main.py
```

`run.py`는 모든 출력을 터미널과 `output_log.txt`에 동시에 기록합니다.

### 저장된 입력 파일로 재현 실행

```bash
python run.py main.py --input player_input.txt
```

> ⚠️ `event.bin` (사건관련정보 + 임무 정답 파일)은 프로젝트 폴더에 함께 있어야 합니다.
> 이 파일은 교수님께서 배포하신 파일이며, 게임 시작 시 `data.py`에서 자동으로 로드합니다.

## 🎮 게임 명령어

| 명령어 | 설명 |
|---|---|
| `상태` | 잔액, HP, 현재위치, 동서남북 출력 |
| `동` / `서` / `남` / `북` | 해당 방향으로 이동 |
| `구매` | 현재 장소에서 구매 (가능한 경우) |
| `판매` | 현재 장소에서 판매 (가능한 경우) |
| `임무` | 현재 장소에서 임무 진행 (가능한 경우) |
| `임무목록` | 현재 가지고있는 임무 목록 출력 |
| `가방` | 가방 안의 물건 확인 및 사용 |
| `난이도` | 난이도 변경 (보통 / 어려움) |
| `저장` | 게임 상태를 파일에 저장 |
| `불러오기` | 저장된 파일에서 게임 상태 불러오기 |
| `종료` | 게임 종료 |

## 🏗️ 클래스 구조

### `Player` (state.py)
주인공의 상태와 동작을 관리합니다.
- `move(direction)`: 동서남북 이동, HP 감소, 장소별 사건정보·상호작용 출력
- `print_status()`: 잔액 / HP / 현재위치 / 동서남북 출력

### `Place` (place.py)
각 장소를 표현합니다. 동일한 형식의 인스턴스들을 하나의 클래스로 모듈화했습니다.
- `buy(player)`: 구매 처리 (잔액 확인, 가방 추가)
- `sell(player)`: 판매 처리 (인벤토리 확인, 잔액 추가)
- `handle_quest(player)`: 임무 처리 (정문 / 독수리상 / 본관 / 세브란스 / 이윤재관 분기)
- `get_interactions()`: 가능한 상호작용 목록 반환 (구매 / 판매 / 임무)

### `Quest` (quest.py)
임무 정보를 관리합니다.
- `description`: 임무 설명
- `report_to`: 보고할 장소

## 📦 사건관련정보 (event.bin)

게임 시작 시 `data.py`의 `load_events()` 함수가 `event.bin`을 읽어 다음 정보를 로드합니다.

```python
{
    "events": {
        "노천극장": "아카라카 공연 티켓 암표 거래가 이루어지고 있다.",
        "대강당": "행사 도시락이 상온에 오래 방치되어 식중독 의심 증상이 보고되었다."
    },
    "answers": {
        "교내 부조리 수사": "노천극장",
        "교내 위생사건 수사": "대강당"
    }
}
```

- `events`: 특정 장소에 도착했을 때 출력되는 사건 메시지
- `answers`: 본관 / 세브란스에서 보고할 때의 정답

## 💾 저장 형식

`pickle`로 직렬화된 dictionary:

```python
{
    "player": Player 인스턴스,    # 상태 + 위치 + 임무 + 인벤토리 + row/col
    "environment": dict,         # 시간 등
    "settings": dict,            # 난이도
    "input_history": list        # 지금까지의 모든 입력
}
```

`불러오기`는 현재 폴더 파일 번호 또는 직접 경로 (상대/절대) 모두 지원합니다.
시스템 파일 (`event.bin` 등)은 저장 파일 목록에서 자동으로 제외됩니다.

## 📝 입출력 기록

`io_helper.py`의 `say()` / `get_input()` 래퍼를 통해 모든 입력과 출력이 자동으로 번호와 함께 파일에 기록됩니다.

- `player_input.txt`: 플레이어가 입력한 모든 명령어 (`[1] 동`, `[2] 임무`, ...)
- `game_output.txt`: 게임이 출력한 모든 메시지 (`[1] ...`, `[2] ...`, ...)

이 두 파일은 채점용 재현성 확인을 위해 사용됩니다.

## 🧪 재현성 테스트

`player_input.txt`의 입력을 그대로 다시 넣어 게임을 실행하면, `game_output.txt`와 동일한 출력이 생성됩니다.

```bash
python run.py main.py --input player_input.txt
```

`run.py`는 입력 파일의 각 줄을 순서대로 `input()` 호출에 주입하여 게임을 자동 재생합니다.
